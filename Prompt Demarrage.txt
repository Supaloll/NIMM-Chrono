RÈGLES D'ARCHITECTURE (critiques) :
1. Un seul fichier index.html — pas de framework, pas de build, pas de dépendance externe (sauf l'API du LLM choisi)
2. Tout le stockage actif passe par localStorage — aucune base de données, aucun backend. L'export/import JSON ne fait que lire/écrire ce localStorage à la demande de l'utilisateur ; ce n'est pas un second système de stockage
3. Les sessions de travail (sessions/current) et les historiques de l'Assistant (par contact) sont des clés localStorage séparées et indépendantes — ne jamais les mélanger
4. Le calcul du quota hebdomadaire (getWeekTotalMs) est la seule source de vérité pour les heures — jamais recalculé ailleurs ni estimé par le LLM
5. Le service worker ne doit jamais mettre en cache les appels vers les API de LLM (Deepseek/Claude/Gemini selon le provider actif) — uniquement les fichiers statiques de l'appli
